# AI内容创作工作流 - 架构说明

## 🎯 目标

自动化AI领域内容创作，覆盖从热点发现到数据优化的全流程。

## 🤖 代理分工

| 代理 | 角色 | 职责 |
|------|------|------|
| **Hermes** | 主控 + 素材生成 | 抓取热点、选题、生成初稿、配图、发布、数据追踪 |
| **Claude Code** | 深度加工 | 技术解读、数据验证、代码示例、内容结构优化 |
| **Windsurf** | 页面构建 | 内容预览页、响应式设计、SEO优化、分享链接 |
| **WorkBuddy** | 团队审核 | 质量把关、品牌调性检查、合规审查、最终确认 |

## 🔄 工作流特性

### 1. 检查点机制
- **自动暂停**: 在步骤7（团队审核）自动暂停
- **状态保存**: 完整保存当前进度到 `.workflow/` 目录
- **灵活恢复**: 审核通过后从检查点继续执行

### 2. 错误处理
- **自动重试**: 关键步骤（热点抓取）失败自动重试3次
- **指数退避**: 重试间隔递增，避免频繁请求
- **优雅降级**: 单步骤失败可选择跳过或终止

### 3. 多代理协作
- **职责清晰**: 每个代理有明确分工
- **数据流转**: 上一步输出自动成为下一步输入
- **上下文保持**: 全程保持完整上下文

### 4. 数据驱动
- **性能追踪**: 自动追踪每篇内容表现
- **反馈闭环**: 数据反馈自动优化下期内容
- **持续迭代**: 基于数据不断优化策略

## 📝 注意事项

1. **合规性**: 确保内容不涉及敏感话题
2. **版权**: 引用他人内容需注明来源
3. **频率**: 避免过度发布导致账号降权
4. **互动**: 及时回复评论，提升账号活跃度
5. **数据**: 定期备份数据，防止丢失
